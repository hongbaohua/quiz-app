12. 為何在SAP S/4HANA版本，許多應用程式(application code)必須重寫(re-written)？

    (A) 因為由ABAP代碼無法在 S/4HANA平台使用

    (B) 因為要讓確保在S/4HANA的效能得以發揮

    (C) 因為S/4HANA只能用JAVA開發

    (D) 因為S/4HANA架構比較複雜


解答與詳細解析

1. (A) SAP ERP 版本時序：R/2(1970s) → R/3(1992) → ERP ECC(2004) → S/4HANA(2015)，對應編號順序為 1-4-2-3，答案為 1423。

2. (B) On-Premise(內部部署)的特點：用戶自行管理資料(2)、較少的更新周期(4)；SAP 提供維運(1)是 Cloud 部署的特點，每季自動更新(3)也是雲端的特性，故選(B) 24。

3. (C) 在 SAP ERP 中，使用 T-CODE「SEARCH_SAP_MENU」可以搜尋功能表中的功能位置，協助使用者找到忘記路徑的功能交易碼。

4. (A) SAP S/4HANA 以 SAP HANA 記憶體內資料庫(In-Memory Database)為核心，需要大量的記憶體(RAM)才能將所有資料載入記憶體進行即時運算。

5. (A) SAP S/4HANA 的雲端整合生態系包含 SAP Ariba(採購)、SAP Concur(差旅)、SAP Hybris(電商)等 SAP 自家雲端產品；NetSuite 是 Oracle 的 ERP 產品，與 SAP S/4HANA 沒有原生整合。

6. (C) SAP S/4HANA 整合的雲端應用包含 SAP Ariba、SAP Concur、SAP Fieldglass 等；LinkedIn 是微軟旗下的社群平台，不在 SAP S/4HANA 的整合範疇。

7. (D) SAP 資料類型分為組織架構(Org. Data)、主數據(Master Data)、交易數據(Transaction Data)；產品別(Division)屬於組織架構資料，不是主數據；工作中心、固定資產、會計科目均屬主數據。

8. (A) 銷售通路(Distribution Channel)是 SAP 銷售模組的組織層次元素，屬於組織架構(Organizational Data)，用來定義銷售渠道的分類。

9. (C) 報價單(Quotation)是銷售流程中的一個業務單據，屬於交易數據(Transaction Data)，因為每次報價都是一筆獨立的業務交易記錄。

10. (B) 在 SAP ERP 的語言代碼中，繁體中文的代碼為「ZF」；ZH 是簡體中文，TW 和 CN 是國家代碼而非語言代碼。

11. (A) HCM(Human Capital Management，人力資本管理)是 SAP 中負責人力資源管理的模組；EAM 是企業資產管理、CS 是客戶服務、PS 是專案系統。

12. (B) S/4HANA 採用全新的 HANA 記憶體資料庫架構，許多應用程式需要重寫以充分利用 In-Memory 技術、消除冗餘彙總表，確保在新架構上發揮最佳效能；ABAP 仍然可以在 S/4HANA 使用。
