## 解析與補充

1. **(B)** 內部部署（On-Premise）由企業自行控管資料與維運，更新週期較長；雲端版（Cloud）才是由 SAP 維運並每季自動更新。
2. **(A)** SAP 演進史：R/2 (1979) -> R/3 (1992) -> ERP ECC (2004) -> S/4HANA (2015)。
3. **(C)** `SEARCH_SAP_MENU` 是非常有用的搜尋 T-CODE 工具。
4. **(A)** HANA 是記憶體內資料庫（In-Memory Database），強大效能依賴大量記憶體。
5. **(D)** 產品別（Division）是組織架構的一部份，而非主數據。工作中心、資產、會計科目均屬主數據。
6. **(A)** 銷售通路是定義企業銷售區域的基礎架構資料。
7. **(C)** 報價單、訂單、發票等屬於流程執行中產生的交易數據。
8. **(B)** SAP 中繁體中文的語言代碼通常為 ZF。
9. **(C)** SAP 雲端整合包含 Ariba (採購)、Concur (差旅)、Fieldglass (外包管理)、SuccessFactors (HR)等，LinkedIn 非 SAP 原生整合模組。
10. **(A)** HCM (Human Capital Management) 是 SAP 的人資模組。
11. **(A)** NetSuite 是甲骨文（Oracle）的產品，非 SAP 原生雲端整合範圍。
12. **(B)** 為了利用 HANA 記憶體資料庫的並行處理與壓縮特性，程式碼需重寫以優化效能。
13. **(D)** 訂單屬於交易數據（Transaction Data），非組織數據。
14. **(D)** SAP 組織架構由上至下：Client -> Company Code -> Sales Organization -> Distribution Channel -> Division。
15. **(D)** SAP Hybris (現為 SAP Commerce Cloud) 專注於銷售、行銷與電商整合。
16. **(C)** 本地端（On-Premise）硬體應建置於企業內部。
17. **(A)** S/4HANA 為了效能優化，通常採用反正規化（De-normalization）或簡化的資料模型。
18. **(A)** Transactional Apps（交易類）用於執行任務；Analytical Apps（應用/分析類）用於視覺化。
19. **(D)** FI（財務會計）屬於 Finance 類別，而非 Logistics 類別。
20. **(D)** 在功能視窗中，需加上 `/N` 或 `/O` 前綴才能輸入新的 T-Code。
21. **(C)** 小數點格式通常是在「使用者設定 (User Profile)」中設定，而非 GUI Options。
22. **(A)** 貿易品（Trading Goods, HAWA）是買賣業直接買進並賣出的物料。
23. **(A)** 用戶端（Client）是 SAP 系統中的最高組織層級。
24. **(A)** 銷售組織（Sales Organization）負責銷售與分銷。
